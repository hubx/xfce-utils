Introduction

Le paquet xfce-utils fournit une barre des tâches, une fenêtre "À propos", et une fenêtre de lancement, ainsi que plusieurs scripts très utiles qui sont aussi utilisé par d'autres composants de Xfce 4, tels le panneau et le menu du bureau.

 

La barre des tâches

La barre des tâches montre les applications en cours de fonctionnement. Le comportement de la barre des tâches peut être modifié depuis le gestionnaire de paramètres. Exécutez xfce-setting-show taskbar, ou lancez le gestionnaire de paramètres depuis le panneau.

Figure 1. La fenêtre de configuration de la barre de tâches

Depuis cette fenêtre de configuration, vous pouvez choisir la position de la barre des tâches, en haut ou enn bas de l'écran. Vous pouvez régler sa hauteur et sa largeur en utilisant les deux curseurs, et selectionner un alignement gauche, centré ou droit, si la largeur de la barre des tâches est inférieure à celle de votre écran. Une autre option permet de sélectionner le masquage automatique.

Les options relatives aux tâches permettent de personnaliser le comportement de la barre des tâches vis à vis des applications ouvertes. Vous pouvez choisir de voir les applications de tous les espaces de travail, ou seulement de l'espace courant, de grouper systématiquement les tâches ou seulement quand la barre des tâches est pleine, et d'afficher les noms des applications ou bien seulement leurs icônes.

En plus de la liste des tâches, la barre des tâches propose un changeur de bureaux graphique et une zone de notification (ou "system tray"). Les deux peuvent être désactivées depuis le gestionnaire des paramètres.

 

La fenêtre de lancement

Figure 2. La fenêtre de lancement

Le fenêtre de lancement permet de démarrer des applications. Ces applications peuvent être lancées dans un émulateur de terminal (xterm par défaut, voir le script xfterm4 ci-dessous). Par défaut dans Xfce 4, le raccourci Alt+F2 ouvre la fenêtre de lancement.

 

La fenêtre À propos

Figure 3. La fenêtre À propos

La fenêtre À propos est utilisée pour afficher les informations concernant Xfce 4, dont la liste des composants, une page de crédits pour toutes les personnes qui ont aidé à créer Xfce 4, et les informations concernant les licences utilisées.

 

Scripts

startxfce4

Le script startxfce4 permet le lancement d'une session Xfce 4 depuis la console. Il ouvre une session comportant la barre des tâches, le panneau, le gestionnaire de bureau et le gestionnaire de fenêtres.

Tous les programmes, ou liens symboliques vers des programmes, présents dans ~/Desktop/Autostart/ seront lancés par startxfce4 au démarrage.

Pour personnaliser le comportement de startxfce4, copiez le fichier ${sysconfdir}/xfce4/xinitrc dans votre répertoire ~/.config/xfce4/ et éditez-le. Si vous avez fait l'installation depuis le code source, ${sysconfdir} est par défaut /usr/local/etc; pour les paquets binaires, c'est le plus souvent /etc.

Depuis l'inclusion d'un gestionnaire de session dans Xfce 4.2, le meilleur moyen de modifier le comportement au démarrége et d'utiliser l'option "Sauvegarder la session" dans la fenêtre de dialogue de fin de session.

xfhelp4

Ce script est utilisé pour ouvrir dans un navigateur la documentation de Xfce 4 installée sur votre système. Par défaut, il essaye plusieurs navigateurs tels firefox, galeon ou konqueror, mais vous pouvez changer cela en modifiant la variable d'environnement $BROWSER. Par exemple, placez cette ligne dans votre fichier ~/.bash_profile: 

export BROWSER="galeon -w"

Vous devrez vous déconnecter de votre shell pour que cela fonctionne.

Si une version de la documentation est disponible dans votre langue, xfhelp4 l'ouvrira à la place de la documentation originale en anglais.

xflock4

Ce script est utilisé par le panneau pour verrouiller l'écran en utilisant xscreensaver, si ce programme est disponible, ou xlock à défaut.

xfmountdev4

xfmountdev4 /dev/mountpoint montera le point de montage correspondant et ouvrira xffm à cet emplacement. Quand vous fermerez xffm, le système de fichiers sera démonté. Notez que vous pouvez aussi utiliser xffstab4 fourni avec xffm pour cela.

xfterm4

Ce sript utilise la variable d'environnement $TERMCMD en tant que commande de lancement d'un terminal. À défaut, il lance xterm. Ceci est utile dans le panneau, où vous pouvez glisser/déposer des fichiers qui seront alors ouverts avec l'application appropriée en mode texte (less pour les fichiers texte, lynx pour les fichiers html).
