Introduction

Le paquetage xfce-utils fournit une barre des tâches, la fenêtre "A propos" de XFce 4, une fenêtre de lancement d'applications et plusieurs scripts utiles qui sont aussi utilisés par d'autres composants de XFce 4 tels le panneau et le menu du bureau

 

La barre des tâches

La barre des tâches est en haut ou en bas de l'écran et affiche les programmes en cours. Son comportement peut être modifié à partir du gestionnaire de paramètres. Lancez xfce-setting-show taskbar ou démarrez le gestionnaire de paramètres à partir du panneau. Outre la liste des programmes, la barre des tâches possède aussi un affichage graphique des bureaux et une zone de notification (aussi appelée system tray). Les deux peuvent être inactivés à partir de la gestion des paramètres.

 

Run Dialog

Figure 1. La fenêtre de lancement d'applications

La fenêtre de lancement d'applications permet à l'utilisateur de lancer des programmes. Les programmes peuvent être lancés dans un émulateur de terminal (par défaut, xterm, voir le script xfterm4 ci-dessous) below).

Dans l'environnement XFce 4 par défaut, la combinaison de touches ouvre la fenêtre de lancement d'applications.

 

Le fenêtre "À propos"

Figure 2. Dialogue A propos

La fenêtre "À propos" est utilisée pour afficher des informations sur XFce 4, comprenant une liste des composants, une page crédits pour tous ceux qui ont participé à la création de XFce 4 and les informations légales.

 

Scripts

startxfce4

Startxfce4 est un script pour lancer une session XFce 4 depuis la console. Il vous procure une session comprenant une barre des tâches et un panneau et lance les gestionnaires de fenêtres et de bureau.

Tous les programmes ou liens symboliques vers des programmes inclus dans ~/Desktop/Autostart/ sont démarrés par startxfce4 au lancement.

Pour personnaliser le comportement de startxfce4, recopiez le fichier ${sysconfdir}/xfce4/xinitrc dans votre répertoire ~/.xfce4/ personnel et modifiez ce fichier. Si vous installez à partir des sources, ${sysconfdir} est par défaut : /usr/local/etc ; à partir d'un paquetage binaire, ${sysconfdir} est souvent /etc. Ce fichier doit être exécutable : 

chmod u+x ~/.xfce4/xinitrc

xfhelp4

Ce script est utilisé pour afficher la documentation de XFce 4 dans un navigateur. Par défaut, il utilise mozilla, mais vous pouves modifier ce choix en indiquant la valeur de la variable d'environnement $BROWSER. Pour ceci, ajouter dans votre ~/.bash_profile la ligne suivante : 

export BROWSER="galeon -w"

Vous devrez quitter et redémarrer votre shell pour que ceci prenne effet.

S'il existe une documentation localisée dans votre langue, xfhelp4 utilisera celle-ci à la place de la version anglaise par défaut.

xflock4

Ce script est utilisé par le panneau pour verrouiller l'écran en utilisant xscreensaver s'il est disponible, ou bien, à défaut, xlock.

xfmountdev4

xfmountdev4 /dev/mountpoint monte le point de montage indiqué et ouvre xffm à cet endroit. A la fermeture de la fenêtre xffm, le système de fichiers est démonté. Notez que vous pouvez utiliser, dans la même intention, xffstab4 fourni avec xffm.

xfterm4

C'est un script de terminal. Il utilise la variable d'environnement $TERMCMD comme commande si elle est précisée, sinon, il utilise xterm. Particulièrement utile dans le panneau : vous pouvez glisser des fichiers où des répertoires vers son icône, ce qui permet de les visualiser, en mode console, avec l'application appropriée (less pour le texte, lynx pour le html).
