Installer XFce4

Composants de XFce 4

Les composants de XFce 4 sont fournis sous la forme de paquetages séparés. Actuellement, les modules suivants sont disponibles.

Bibliothèques

Au contraire des précédentes versions, XFce 4 fournit une plateforme de développement consistant en un ensemble de bibliothèques qui offrent des fonctions d'aide et des widgets permettant à un programme de s'adapter facilement à l'environnement XFce 4.

libxfce4util

Bibliothèque avec les fonctions d'aide non graphique.

libxfcegui4

Bibliothèque des widgets, interaction avec le système X.

libxfce4mcs

Bibliothèque de gestion des paramètres, utilisée par la plupart des modules de XFce 4.

Bureau

Ces modules fournissent les fonctions essentielles de l'environnement de bureau.

xfce-mcs-manager

Gestionnaire de paramètres, il offre un centre de contrôle pour la gestion des plugins installés par les autres modules.

xfce-mcs-plugins

Les plugins de gestion de paramètres. Ces plugins offrent des fenêtres de dialogue pour des éléments qui ne font pas partie de XFce 4, tels les paramètres de GTK, de la souris et du clavier.

xfwm4

Le gestionnaire de fenêtres Xfce 4. Gère le placement des fenêtres des applications sur l'écran, fournit le décor des fenêtres et gère les bureaux ou espaces de travail virtuels.

xfce4-panel

Le panneau XFce 4. Fournit les lanceurs de programmes, un changeur d'espace de travail, des menus, etc.

xfdesktop

Le gestionnaire de fond d'écran du bureau. Ce programme installe l'image et/ou la couleur de fond et fournit un menu de bureau. Il comprend aussi deux fenêtres de dialogue pour vous permettre de modifier le fond d'écran ou le nombre d'espaces de travail.

xfce-utils

Scripts et utilitaires essentiels. Fournit la barre des tâches, la fenêtre "À propos" de XFce 4, un fenêtre de lancement d'applications et plusieurs scripts importants, utilisés par les autres paquetages, y compris le panneau. Contient aussi ce guide de l'utilisateur.

xffm

Gestionnaire de fichiers. Permet également de naviguer parmi les dossiers partagés d'un réseau samba.

xfprint

Fenêtre de dialogue pour l'impression. Offre une interface graphique pour l'impression. Comprend xfprint4 et xfprint-manager.

gtk-xfce-engine-2

Moteur de thèmes pour GTK2. Il est facultatif, mais c'est un moteur de thèmes réussi, que vous devriez essayer.

Extras

Il existe nombre de paquetages supplémentaires, ne faisant pas partie du coeur de l'environnement XFce 4, mais offrants d'autres thèmes ou d'autres fonctionnalités.

xfwm4-themes

Décorations de fenêtres pour xfwm4.

xffm-icons

Thèmes d'icônes pour le gestionnaire de fichiers xffm.

xfce4-themes

Thèmes d'icônes pour le panneau XFce 4.

xfce4-iconbox

La boîte d'icônes peut être utilisée en remplacement de la barre des tâches fournie par le paquetage xfce-utils. Lire the section called “Démarrer” pour voir comment modifier le script de démarrage.

xfce4-mixer

Contrôle de volume pour le panneau. Comprend un simple mixer.

xfce4-systray

La zone de notification pour le bureau. Retenez qu'il ne peut y avoir qu'une seule zone en même temps, vous devez retirer la zone de notification de la barre des tâches pour utiliser celle du panneau.

xfce4-toys

Ajouts et utilitaires moins sérieux du panneau. Fournit actuellement une fenêtre de dialogue pour les trucs et astuces / fortunes, et une implémentation de xeyes.

xfce4-trigger-laucher

Offre un bouton commutateur sur le panneau, affichant un état on/off. Lire le fichier README fourni avec le paquetage pour plus d'information.

 

Paquetages binaires

Officiellement, le projet XFce 4 ne fournit que le code source de son environnement de bureau. Cependant des paquetages binaires ont peut-être été réalisés par des contributeurs pour votre OS ou votre distribution.

Martin Loschwitz est le développeur Debian qui maintient les paquetages XFce 4 pour la Debian unstable. Les mises à jour sont rapidement disponibles après une nouvelle version.

Jetez un oeil sur le site XFce Téléchargements pour voir les informations complémentaires sur la disponibilité des paquetages binaires.

Si vous ne trouvez pas de paquetages pour votre OS, ils sont peut-être disponible auprès de votre fournisseur ou distributeur, ou encore vous pouvez installer XFce 4 à partir des sources.

 

Installer à partir des sources

Allez sur le site XFce Téléchargements pour obtenir les archives des sources.

Compiler les modules devrait être aussi simple que décompresser les archives et, à partir du répertoire parent, taper : 

./configure && make && make install

Certains modules peuvent avoir des options de configuration. Vous pouvez les obtenir en tapant : ./configure --help.

La compilation de XFce 4 dépend de pkg-config, GTK+ >= 2.0.6, libxml2 et, pour xffm, de libdbh, qui est disponible sur SourceForge. Si vous installez ceux-ci à partir de paquetages binaires, assurez-vous d'installer aussi les paquetages de développement (paquetages -dev).

Vous pouvez aussi (facultatif) installer librsvg >= 2.2 pour le support des icônes SVG et libstartup-notification pour disposer du curseur "occupé" au chargement des applications qui supportent la notification de démarrage.

Si vous effectuez l'installation avec un préfixe différent de celui de pkg-config, vous devrez renseigner la variable PKG_CONFIG_PATH pour y inclure le chemin vers les fichiers *.pc installés par les bibliothèques de XFce 4 et qui est ${prefix}/lib/pkgconfig. Par exemple : 

export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig

Toutes les archives des sources contiennent un fichier de spécifications pour vous permettre de construire vos propres rpm : 

rpmbuild -ta module-x.y.z.tar.gz

 

Utiliser CVS

Voici les instructions pour obtenir les dernières mises à jour depuis le CVS. Vous pouvez aussi utiliser le lien WebCVS pour consulter les archives.
