Installer Xfce4

Paquets binaires

Le projet Xfce ne met à disposition que le code source de son environnement de bureau. Cependant, des paquets binaires sont proposés par des tiers pour votre système d'exploitation ou votre distribution.

Visitez le site officiel de Xfce pour connaître les liens de téléchargement et obtenir plus d'informations à propos des paquets disponibles. Si vous ne trouvez pas de paquets pour votre distribution, ils sont peut-être disponibles auprès de l'éditeur de votre système, ou bien vous devrez installer Xfce 4 depuis son code source.

 

Installeurs graphiques

Le moyen le plus facile d'installer Xfce depuis les sources est d'utiliser les installeurs graphiques. Documentation et instructions d'utilisation sont disponibles.

 

Compiler depuis le code source

Pour compiler Xfce manuellement, visitez le site officiel de Xfce pour télécharger le code source.

Compiler les composants de Xfce consiste simplement à décompresser les archives et à taper, dans le répertoire parent de chaque composant: 

./configure && make && make install

Certains composants proposent des options de configuration supplémentaires, que vous pouvez découvrir en tapant ./configure --help.

Xfce 4 dépend de pkg-config, GTK+ >= 2.2, libxml2 et, concernant xffm, sur libdbh, qui est disponible sur SourceForge. Si vous avez installé ces dépendances au moyen de paquets binaires, assurez-vous d'avoir installé aussi les paquets -dev correspondants.

De manière optionnelle, vous pouvez installer librsvg >= 2.2, pour bénéficier du support des icônes SVG, et libstartup-notification, pour avoir une pointeur de souris modifié lors du démarrage des applications qui supportent ce standard.

Si vous installez Xfce en utilisant un préfixe différent de celui de pkg-config, vous devrez renseigner la variable d'environnement PKG_CONFIG_PATH pour y inclure le chemin vers les fichiers *.pc installés par les librairies de Xfce 4, ce qui correspond à ${prefix}/lib/pkgconfig. Par exemple: 

export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig

Toutes les archives du code source contiennent un fichier appelé spec qui vous permet de construire votre propre rpm: 

rpmbuild -ta module-x.y.z.tar.gz

 

Installer depuis le serveur CVS

Le code source de Xfce 4 en cours de développement est disponible depuis le serveur CVS.

Visitez cette adresse pour plus d'information sur la façon d'obtenir les dernières mises à jour depuis ce serveur. Vous pouvez a ussi utiliser le WebCVS pour naviguer dans son arborescence.

 

Applications supplémentaires

Le projet Xfce Goodies réunit les contributions à Xfce 4. Il propose notamment d'excellents plugins pour le panneau qui agrémentent l'utilisation de Xfce 4.

Xfmedia est un lecteur multimedia léger pour Xfce, basé sur le moteur xine. C'est un lecteur parfait pour votre environnement.

Terminal est un terminal avancé mais facile à utiliser conçu pour l'environnemnet de bureau Xfce/ Il propose toutes les fonctionnalités que vous êtes en droit d'attendre de la part d'un émulateur de terminal moderne.
